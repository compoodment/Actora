=== CLARIFIED DESIGN READ (PRESERVED WORDING LAYER) ===

Actora is a single continuous universe simulation where the player primarily lives as the current actor, but can zoom across scopes, continue through connected lives, and interact with real economic, political, spatial, and later social systems inside the same save.

Core clarified points:
- one save = one universe simulation
- actor life is still the emotional anchor
- full custom sandbox setup is the default start
- setup is authored, then the simulation runs emergently
- the intended zoom ladder is actor -> city -> country -> world -> star system -> galaxy -> universe
- zoomed-out scopes should still expose actions through real entities/systems where possible rather than detached fake god buttons
- exact inspectable information is allowed throughout the simulation
- the game is one continuous simulation rather than a pile of hard-separated modes
- the focused actor should still progress semi-autonomously when time is skipped
- death is a structural transition, not a fail-state philosophy
- a line can be archived without ending the universe
- continuity can continue through relevant connected actors rather than strict child-only succession
- economy is a pillar
- politics/world conditions are a pillar
- companies should eventually exist as real hybrid simulation things, especially when player-owned or player-relevant
- earliest practical era target is modern + near future
- earliest practical place target is world / country / city
- later social systems should use multiple perception axes rather than a dumb single karma meter

Anti-drift read:
Actora should not quietly collapse back into “deeper BitLife,” “detached god sim,” “spreadsheet world sim with humans taped on,” or vague universe-scale handwaving. The intended shape is actor-anchored, zoomable, continuous, and simulation-real.
