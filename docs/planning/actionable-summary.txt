=== CURRENT CLARIFIED DESIGN READ ===

Actora is now more clearly understood as:
- one continuous universe simulation per save
- primarily actor-played rather than detached-overseer-first
- full custom sandbox setup by default, followed by emergent simulation
- same-save scope zoom from actor -> city -> country -> world -> star system -> galaxy -> universe
- one game rather than a pile of hard-separated “modes”
- exact inspectable information available throughout the simulation
- actor switching allowed through relevant connected actors rather than forced child-only continuation
- death as a structural transition, not a fail-state philosophy
- economy as a pillar, not garnish
- politics/world conditions as a pillar, not garnish
- companies as eventual real hybrid simulation objects, especially when player-owned or player-relevant
- modern + near-future as the earliest practical era target
- world / country / city as the earliest practical place target
- social perception as a later but important multi-axis layer rather than a dumb single karma score

Cleanest one-line read:
Actora is a single continuous universe simulation where the player primarily lives as the current actor, but can zoom across scopes, continue through connected lives, and interact with real economic, political, spatial, and later social systems inside the same save.

=== REAL TENSIONS STILL PRESENT ===

1. Actor-primary fantasy + exact inspectable info everywhere
This remains a valid but unusual combination. It leans toward a playable simulation interface rather than strict in-character uncertainty.

2. Full sandbox default + “real life” fantasy
This is coherent, but only if the game stays emotionally anchored in actor life instead of collapsing into abstract god-sim behavior.

3. Broad continuity after death lowers punishment stakes
This appears intentional. Weight should come from records, continuity, lineage, archive meaning, and universe persistence rather than fail-state drama.

4. Scope remains huge
The vision is coherent, but the execution risk is still “feature landfill with delusions of grandeur” if sequencing discipline slips.

5. Operator/dev tooling temptation is still rising before enough systems exist
This is a real pressure, but a full dev mode right now could become premature tooling fluff if it gets built before enough repeated testing pain justifies a reusable operator layer.

6. The spatial model is ahead of the spatial mutation boundary
The repo now stores more honest spatial identity, but future systems would still be too tempted to poke actor fields directly unless a cleaner world-owned mutation seam arrives soon.
