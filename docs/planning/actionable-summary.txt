=== CURRENT WORKING POSITION ===

CompLife now has a materially stronger early backbone, and verified repo truth is through v0.25.0:
- world-owned actor registry
- world-owned links
- structured place registry
- current place and residence separation
- dedicated identity-generation layer
- centralized event outcome application
- world-owned record storage
- world-owned record query helpers
- explicit record-shape normalization
- explicitly human-backed world-owned constructor seam (`create_human_actor(...)`)
- structured snapshot output via `Human.get_snapshot_data(...)` with shell-owned snapshot rendering in `main.py`
- cleaned monthly event result contract, with `outcome.stat_changes` as the single authoritative mutation payload and no redundant top-level `stat_changes`
- grounded human-only event coverage extending across Infant, Child, Teenager, Young Adult, Adult, and Elder
- explicit startup-family semantic metadata for origin, caregiving, and bootstrap provenance on current family links in `World.links`
- narrow world helper access for semantic family lookup while preserving compatibility parent lookup
- a more honest human-scoped monthly event seam through `HUMAN_MONTHLY_EVENTS`, `_get_human_eligible_events(...)`, `get_human_monthly_event_from_lifecycle(...)`, and compatibility `get_monthly_event(...)`
- a structured identity-generation context seam through `prepare_parent_identity_context(...)`, `generate_parent_identity_from_context(...)`, and compatibility `generate_parent_identity(...)`
- a world-owned authoritative simulation-step boundary through `World.simulate_advance_turn(...)`
- explicit invalid-actor boundary guarding on the world-owned simulation-step boundary
- explicit unknown-stat mutation guarding in `Human.modify_stat(...)`

That means the project is no longer mainly fighting:
- for basic structural honesty in the early backbone,
- for the obvious practical event-silence gap after Child,
- for startup-family semantic collapse,
- for the event seam sounding broader than current repo truth,
- for identity generation being only a loose-argument call path with no context-shaped seam at all,
- for the simulation-step boundary being only half-owned, or
- for silent swallowing of unsupported stat mutations.


=== CURRENT PLANNING READ ===

There is no equally obvious backbone patch after v0.25.0.

The right next move is:
**a deliberate v0.26.0 decision pass**

Why:
- the last cluster of loud, narrow backbone/event/family/identity/step-boundary/mutation fixes is now landed
- nothing of equal architectural weight has replaced them yet
- the strongest visible remaining pressures are now runtime/input hardening and future-expansion footgun candidates
- blindly inventing another broad patch would be weaker than choosing deliberately

So the current question is no longer:
- “what is the next major foundation seam?”

The current question is:
- “is v0.26.0 a tiny runtime-boundary patch, or should the project deliberately pause and keep planning?”


=== LEADING CANDIDATE BUCKETS FOR V0.26.0 DECISION PASS ===

1. CLI input boundary hardening
Read:
- raw `input()` paths still fail noisily on `EOFError` / `KeyboardInterrupt`
- interactive happy-path works, but non-happy-path exits are still rough

Use:
- strongest current small candidate if v0.26.0 stays tiny and runtime-focused
- do not let it broaden into general shell cleanup

2. Singleton startup actor IDs
Read:
- startup still uses hardcoded singleton IDs like `"mother"` and `"father"`
- not a current problem, but a future expansion tripwire

Use:
- keep parked unless startup scope broadens
- do not promote early

3. Entity registry vs actor view layering
Read:
- still a real long-term guardrail
- still too easy to promote too early

Use:
- keep parked unless a sharply bounded seam appears
- do not promote just because it sounds important

4. Identity generation realism / context-awareness
Read:
- seam is better after v0.23.0
- actual realism is still blocked by missing place/era/context maturity

Use:
- keep parked unless deliberately chosen later
- do not build fake context sophistication
